LLM Service 架构

统一 LLM 调用与审计服务。独立 FastAPI 进程（:8900），集中管理所有 LLM 调用的提交、执行、重试、结果解析和审计记录。Mining 异步批量，Serving 同步在线。

02

数据流

同步 execute（Serving）

Client.execute()
  → POST /api/v1/execute
  → Service.submit()：创建任务
  → 原子认领（atomic claim）
  → Executor.run()：
      构建 messages → Provider.chat()
      → Parser.parse() → 记录 attempt+result+event
  → 构建 ExecuteResponse 直接返回

异步 submit（Mining）

Client.submit()
  → POST /api/v1/tasks
  → Idempotency 检查 → 创建任务
  → 返回 task_id

后台 Worker 循环：
  → TaskManager.claim() 获取任务
  → Executor.run() 执行（含重试）
  → 持久化到数据库

Client.get_result(task_id) 轮询获取

任务生命周期

queued
→
running
→ ✓
succeeded
或
failed
→ 重试
dead_letter
状态含义转换条件
queued等待 Worker 认领submit() 创建；failed 重试回此状态
runningWorker 正在执行claim() 原子认领；设置 lease 过期时间
succeeded成功完成Executor 完成且 parse_status=succeeded
failed本次尝试失败Provider 错误或解析失败；未达 max_attempts 则回 queued
dead_letter永久失败attempt_count ≥ max_attempts（默认 3）
cancelled手动取消POST /api/v1/tasks/{id}/cancel

双模式调用

Mining 异步批量

适合批量处理：section_summary、question_gen、entity_extract

# 批量提交
task_ids = [await client.submit(
    caller_domain="mining",
    pipeline_stage="question_gen",
    template_key="mining-question-gen",
    input={"title": seg.title, "content": seg.text},
    idempotency_key=f"seg-{seg.id}-qgen",
) for seg in segments]

# 批量轮询
results = [await client.get_result(tid) for tid in task_ids]

Serving 同步在线

适合实时查询：query_rewrite、intent_extract、rerank

result = await client.execute(
    caller_domain="serving",
    pipeline_stage="query_rewrite",
    template_key="serving-query-understanding",
    input={"query": user_query},
    expected_output_type="json_object",
)
rewritten = result["result"]["parsed_output"]

07

重试与容错

错误类型来源处理
timeoutProvider 响应超时重试，指数退避
connection_error网络不可达重试，指数退避
rate_limited429 Too Many Requests重试，更长退避
server_error5xx重试，指数退避
client_error4xx（非 429）不重试，直接 failed
退避公式：delay = base ^ attempt_no，上限 backoff_max。默认 3 次重试。所有尝试都记录在 attempts 表中，包括 token 用量和延迟。
降级策略：同步执行超时不直接失败，任务保持 running 状态，由 lease_recovery 机制后续处理。避免客户端超时导致任务丢失。
