#### 中间模型

```python
class EntityRef(BaseModel):
    type: str          # command / feature / term / alarm / network_element
    name: str
    normalized_name: str

class NormalizedQuery(BaseModel):
    original_query: str
    intent: str                    # command_usage / troubleshooting / concept_lookup / procedure / general
    entities: list[EntityRef]
    scope: dict                    # 通用 facets dict
    keywords: list[str]            # 去停用词后的关键词
    desired_roles: list[str]       # 偏好的 semantic_role

class QueryPlan(BaseModel):
    intent: str
    keywords: list[str]
    entity_constraints: list[EntityRef]
    scope_constraints: dict        # 通用 facets
    desired_roles: list[str]
    desired_block_types: list[str]
    budget: RetrievalBudget
    expansion: ExpansionConfig

class RetrievalBudget(BaseModel):
    max_items: int = 10            # 最终返回的最大 context items 数
    max_expanded: int = 20         # relation expansion 后最大条目数
    recall_multiplier: int = 5     # over-recall 倍数

class ExpansionConfig(BaseModel):
    enable_relation_expansion: bool = True
    max_relation_depth: int = 1    # 只扩展一层
    relation_types: list[str] = ["previous", "next", "same_section", "section_header_of"]
```

#### 响应模型：ContextPack

```python
class ContextQuery(BaseModel):
    original: str
    normalized: str
    intent: str
    entities: list[EntityRef]
    scope: dict
    keywords: list[str]

class ContextItem(BaseModel):
    id: str
    kind: str                      # retrieval_unit / raw_segment
    role: str                      # seed / context / support
    text: str
    score: float
    title: str | None = None
    block_type: str = "unknown"
    semantic_role: str = "unknown"
    source_id: str | None = None   # 指向 sources 中的 ID
    relation_to_seed: str | None = None  # context items 才有，说明与 seed 的关系
    metadata: dict = {}

class SourceRef(BaseModel):
    id: str
    document_key: str
    title: str | None = None
    file_type: str = "unknown"
    relative_path: str | None = None
    section_path: list | None = None
    source_offsets: dict = {}
    facets: dict = {}              # 从 facets_json 读取
    processing_profile: dict = {}

class Issue(BaseModel):
    type: str    # no_result / low_confidence / ambiguous_scope / partial_context / source_unparsed
    message: str
    detail: dict = {}

class ContextPack(BaseModel):
    query: ContextQuery
    items: list[ContextItem]
    sources: list[SourceRef]
    issues: list[Issue]
    suggestions: list[str] = []
    debug: dict | None = None
```

## Task 3: 重写 Repository

重写 `repositories/asset_repo.py`，面向 v1.1 新表。

### 类结构

```python
class AssetRepository:
    def __init__(self, db: aiosqlite.Connection): ...

    async def ensure_active(self) -> str | None
        """确认存在 active publish_version，返回其 ID。无 active 则返回 None。"""

    async def search_retrieval_units(self, plan: QueryPlan) -> list[dict]
        """主检索入口。"""

    async def expand_context(self, segment_ids: list[str], relation_types: list[str]) -> list[dict]
        """沿 raw_segment_relations 扩展关联 raw_segments。"""

    async def resolve_sources(self, unit_ids: list[str]) -> list[dict]
        """批量查 retrieval_unit -> raw_segment -> raw_document，返回 source 信息。"""
```

### 方法详细设计

#### `ensure_active()`

```text
SELECT id FROM asset_publish_versions WHERE status = 'active' LIMIT 1
```

v1.1 资产表不带 `publish_version_id`，不需要按行过滤。只做全局存在性检查。

#### `search_retrieval_units(plan)`

两阶段：召回 + 评分截断。

**召回阶段：**

1. 先尝试 FTS5：
   ```sql
   SELECT ru.*, fts.rank
   FROM asset_retrieval_units_fts fts
   JOIN asset_retrieval_units ru ON ru.id = fts.retrieval_unit_id
   WHERE asset_retrieval_units_fts MATCH ?
   ORDER BY fts.rank
   LIMIT ?
   ```
   MATCH 内容由 keywords 组合（OR 连接）。

2. FTS 无结果时 fallback LIKE：
   ```sql
   SELECT * FROM asset_retrieval_units
   WHERE search_text LIKE ? OR search_text LIKE ? ...
   LIMIT ?
   ```

3. recall_limit = max(plan.budget.max_items * plan.budget.recall_multiplier, 50)

**scope 过滤（Python 侧）：**

对召回结果按 `facets_json` 过滤：
- 如果 plan.scope_constraints 有某个 key（如 product），而 retrieval_unit 的 facets 里没有对应值 → 保守视为不匹配
- 如果 plan.scope_constraints 有某个 key，retrieval_unit 也有 → 检查是否有交集
- 如果 plan 无 scope 约束 → 不过滤

**评分截断：**

```python
def _score(result, keywords, desired_roles, desired_block_types):
    text = (result["search_text"] + " " + result.get("text", "")).lower()
    hit_count = sum(1 for kw in keywords if kw.lower() in text)
    keyword_score = hit_count / len(keywords) * 10 if keywords else 0
    role_bonus = 2.0 if result["semantic_role"] in desired_roles else 0
    block_bonus = 1.0 if result["block_type"] in desired_block_types else 0
    weight = float(result.get("weight", 1.0))
    return keyword_score + role_bonus + block_bonus + weight
```

按 score 降序截断到 `plan.budget.max_items`。

**entity 加权（可选增强）：**

如果 plan 有 entity_constraints，且 retrieval_unit 的 `entity_refs_json` 包含匹配的 entity → 额外加 5 分。

#### `expand_context(segment_ids, relation_types)`

```sql
SELECT rs.*, r.relation_type, r.distance
FROM asset_raw_segment_relations r
JOIN asset_raw_segments rs ON rs.id = r.target_segment_id
WHERE r.source_segment_id IN (?, ?, ...)
  AND r.relation_type IN (?, ?, ...)
```

返回扩展出的 raw_segments，附带 relation_type 和 distance。

#### `resolve_sources(unit_ids)`

```sql
SELECT
    ru.id as unit_id,
    ru.source_refs_json,
    rs.id as segment_id, rs.raw_text, rs.section_path, rs.section_title,
    rs.block_type, rs.semantic_role, rs.structure_json, rs.source_offsets_json,
    rs.entity_refs_json,
    rd.id as doc_id, rd.document_key, rd.title as doc_title,
    rd.file_type, rd.relative_path, rd.scope_json, rd.processing_profile_json
FROM asset_retrieval_units ru
LEFT JOIN asset_raw_segments rs ON rs.id = ru.target_id
LEFT JOIN asset_raw_documents rd ON rd.id = rs.raw_document_id
WHERE ru.id IN (?, ?, ...)
```

一条 SQL 批量解决，不 N+1。

### 验收标准

- 所有查询使用参数化 `?`，无 SQL 注入
- LIKE 值通过 `_escape_like()` 转义
- FTS 无结果时 fallback 到 LIKE
- scope 过滤遵循容错原则：字段缺失时不崩溃

## Task 4: 重写 Assembler

重写 `application/assembler.py`，从 repo 返回的 dict 组装 ContextPack。

### 类结构

```python
class ContextPackAssembler:
    def assemble(
        self,
        normalized: NormalizedQuery,
        plan: QueryPlan,
        units: list[dict],           # search_retrieval_units 结果
        expanded: list[dict],        # expand_context 结果
        sources: list[dict],         # resolve_sources 结果
    ) -> ContextPack
```

### 组装逻辑

#### 1. seed items

从 `units`（retrieval_units 搜索结果）构造：

```python
ContextItem(
    id=unit["id"],
    kind="retrieval_unit",
    role="seed",
    text=unit["text"],
    score=unit["_score"],
    title=unit.get("title"),
    block_type=unit.get("block_type", "unknown"),
    semantic_role=unit.get("semantic_role", "unknown"),
    source_id=unit["_source_ref_id"],  # 通过 target_id 关联
)
```

#### 2. context items

从 `expanded`（relation 扩展的 raw_segments）构造：

```python
ContextItem(
    id=seg["id"],
    kind="raw_segment",
    role="context",
    text=seg["raw_text"],
    score=0,  # 扩展项不打分
    title=seg.get("section_title"),
    block_type=seg.get("block_type", "unknown"),
    semantic_role=seg.get("semantic_role", "unknown"),
    source_id=seg["raw_document_id"],
    relation_to_seed=seg["relation_type"],
)
```

#### 3. sources

从 `sources` 去重聚合（多个 item 可能来自同一文档）：

```python
SourceRef(
    id=doc["doc_id"],
    document_key=doc["document_key"],
    title=doc["doc_title"],
    file_type=doc.get("file_type", "unknown"),
    relative_path=doc.get("relative_path"),
    section_path=_safe_json_parse(doc.get("section_path")),
    source_offsets=_safe_json_parse(doc.get("source_offsets_json")),
    facets=_safe_json_parse(doc.get("scope_json")),  # document 级 scope
    processing_profile=_safe_json_parse(doc.get("processing_profile_json")),
)
```

#### 4. issues

检测条件：

| issue type | 触发条件 |
|---|---|
| `no_result` | seed items 为空 |
| `low_confidence` | seed items 最高 score < 2.0 |
| `ambiguous_scope` | plan.scope 为空且多个 seed 来自不同 facets 域 |
| `partial_context` | 有 seed 但没有 expanded context |

#### 5. suggestions

- 如果 `no_result`：返回 ["尝试使用更具体的关键词", "缩小查询范围"]
- 如果 `ambiguous_scope`：从 seed items 的 facets 中提取可能的维度，建议补充

#### 6. JSON 容错

所有 JSON 解析统一走 `_safe_json_parse(value, default)`：
- None → default
- str → json.loads，失败 → default
- 已解析对象 → 原样返回

### 验收标准

- 输出结构严格符合 ContextPack 模型
- 无 canonical/EvidencePack 残留
- 所有 JSON 字段容错解析

## Task 5: 重写 API 层

重写 `api/search.py`。

### 端点

只保留一个检索端点：

```python
@router.post("/api/v1/search")
async def search(request: SearchRequest, repo=Depends(get_repo)) -> ContextPack
```

删除 `/api/v1/command-usage`。

### 管线

```python
async def search(request, repo):
    # 1. 确认 active
    active_id = await repo.ensure_active()
    if not active_id:
        raise HTTPException(503, "No active publish version")

    # 2. Query Understanding
    normalized = QueryNormalizer().normalize(request.query, request.scope, request.entities)

    # 3. Build Plan
    plan = build_plan(normalized)

    # 4. Retrieve
    units = await repo.search_retrieval_units(plan)

    # 5. Expand context
    segment_ids = [u["target_id"] for u in units if u.get("target_type") == "raw_segment"]
    expanded = await repo.expand_context(segment_ids, plan.expansion.relation_types) if segment_ids else []

    # 6. Resolve sources
    all_unit_ids = [u["id"] for u in units]
    sources = await repo.resolve_sources(all_unit_ids) if all_unit_ids else []

    # 7. Assemble
    pack = ContextPackAssembler().assemble(normalized, plan, units, expanded, sources)

    # 8. Debug
    if request.debug:
        pack.debug = {"plan": plan.model_dump(), "active_version_id": active_id}

    return pack
```

### 验收标准

- `/command-usage` 不存在
- `/search` 返回 ContextPack
- 503 在无 active version 时正确触发
- debug=true 时 debug 字段填充

### normalizer.py

```python
class QueryNormalizer:
    def normalize(self, query: str, scope_override: dict | None = None,
                  entity_override: list[EntityRef] | None = None) -> NormalizedQuery:
        intent = self._detect_intent(query)
        entities = self._extract_entities(query)
        scope = self._extract_scope(query)
        keywords = self._extract_keywords(query)
        desired_roles = self._get_desired_roles(intent)

        # 显式传入的优先
        if scope_override:
            scope = {**scope, **scope_override}
        if entity_override:
            entities = entity_override

        return NormalizedQuery(
            original_query=query,
            intent=intent,
            entities=entities,
            scope=scope,
            keywords=keywords,
            desired_roles=desired_roles,
        )
```

**与旧版本的关键区别：**
- 不再有 `variant_policy`、`missing_constraints`、`scope_variant` 等概念
- scope 不再是固定 `QueryScope(products=, versions=, ...)` 结构，改为通用 dict
- 不再有 `desired_semantic_roles` → 改名 `desired_roles`

### build_plan()

独立函数，从 NormalizedQuery 生成 QueryPlan：

```python
def build_plan(normalized: NormalizedQuery) -> QueryPlan:
    return QueryPlan(
        intent=normalized.intent,
        keywords=normalized.keywords,
        entity_constraints=normalized.entities,
        scope_constraints=normalized.scope,
        desired_roles=normalized.desired_roles,
        desired_block_types=_block_types_for_intent(normalized.intent),
        budget=RetrievalBudget(),
        expansion=ExpansionConfig(),
    )
```

### 验收标准

- 输出 `NormalizedQuery`，不依赖任何旧模型
- scope 是通用 dict，不是固定 Pydantic 模型
- 无 canonical/EvidencePack 残留
